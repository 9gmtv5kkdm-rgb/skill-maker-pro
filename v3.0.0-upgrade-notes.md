# v3.0.0 升级笔记（2026-07-07）

## 升级触发

人类Nan要求元技能对比**自制的写AI痕迹类技能**和**虾评 Top 10 榜单**。结论：

> **元技能哲学深度 ≈ Top 10，但"被找到 / 30 秒看懂 / 立刻试一个"这三个临门一脚上输了。**

更刺痛的是：**元技能自己触犯了自己教别人的所有反模式**。这是"教别人不做的，自己在做"的经典讽刺。

**v3.0.0 的核心任务** = **言行一致**——元技能自己必须先通过自己定的 PF 质量门。

## 经验教训

1. **元技能不是"教别人"**——它必须"自己先做对"
2. **质量门不是"建议"**——它是"机制"
3. **5 分钟路过测试**比"评测 Agent 跑分"更接近用户真实场景
4. **双层结构是基础**——单文件不可能同时服务"5 分钟"和"1 年"
5. **AP 形式化**比"我觉得有问题"更可执行
6. **发布前 dry-run 完整 PF**——避免假数据自报（v3.0.0 踩坑）
7. **版本号必须诚实**——未达标不给 v3.0 名号（v2.9.3 教训）
8. **专家团并审**——6 路 push-based 不轮询，P0 全修才能上 Top 5
