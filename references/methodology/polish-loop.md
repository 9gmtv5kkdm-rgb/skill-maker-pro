# 门面打磨与发布后运营循环（详版）

> 本节为 SKILL.md「门面打磨与发布后运营循环」的展开细节。按双层结构（SKILL.md = 索引门面，厚方法论下沉 references/）从正文收编至此，内容不删、仅移位。

来源：技能门面打磨循环（skill-polish-loop，原独立技能）实战沉淀的「治本→改造→评估→反哺→再改」五阶段滚动闭环。它解决一个真问题：**功能硬的技能没人用 / 发了没下载，根因是门面差或门面同质化**。门面不是装饰，是"路人怎么发现并想用"的开关。v3.0.10 将其精华收编为元技能内置方法论，原技能退役留档。

## 五阶段（每轮 R 迭代）

| 阶段 | 做什么 | 关键动作 | 产出 |
|---|---|---|---|
| **S1 治本** | 改元技能，加门面规范 | 在元技能加 PF（如 PF11 门面质检、PF11f 去模板化、L3 唤醒） | 元技能新版本 |
| **S2 改造** | 用新规批量改技能 | 补 displayName / 用户原声触发词 / 首屏钩子 / 收益型 description / 放宽链路 | 技能门面达标 |
| **S3 评估** | 多维度多角色评估 | 全量扫描量化 + 抽样三角色（爆款制作者/普通路人/领域专家）× 三层次（L1门面/L2功能/L3复利） | 评估报告 + 新痛点清单 |
| **S4 反哺** | 分析痛点，完善元技能 | 把评估发现的盲点写成新 PF 或补强现有 PF | 元技能再升级 |
| **S5 再改** | 用改进元技能再改一轮 | 针对新痛点精修（如去模板化：每技能写独有具体收益钩子） | 下一轮基线更高 |

循环终止：S3 三角色评估无新痛点，或用户叫停。

## 踩坑禁区（来自 R1–R7 实战，刻进肌肉记忆）

- **只插不删**：批量改造技能时只允许"插入新门面字段/小节"，绝不用 `re.sub` 等整体替换吞掉相邻小节——曾因此误删起源铭文与不可变核心。
- **版本号三重一致**：SKILL.md YAML / §9 演进史 / evolution-state.json 的 version 必须一致，否则判 P0。
- **时间戳副本目录排除**：扫描脚本需排除 `vX.Y.Z_YYYYMMDD_HHMMSS` 这类时间戳副本目录，否则重复计数假阳性。
- **frontmatter↔正文双轨**：`triggers:` 字段若是系统指令（如 `contract-intake 锁定 review_path=`）是链路锁死真根因，必须改用户原声，且 frontmatter 与正文小节同步改。
- **去模板化是硬门槛**：批量拉齐门面后必查同质化——同类技能钩子/触发词/description 不得雷同；每个技能写独有具体收益（含场景/数字/对比）。
- **先打快照再改造**：批量改前必须本地快照备份，否则无回滚点。

## 升级安全规则（沿用）

S1/S4 改元技能需人类确认版本与 PF 表述；L3 改子技能 SKILL.md 必须人类确认（不可变核心第 2 条）。
