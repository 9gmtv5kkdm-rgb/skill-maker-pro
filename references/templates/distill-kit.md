# 子技能自我蒸馏赋权标准（v3.0.2 新增）

> **核心理念：母体赋权，子体用权。** 元技能**不读取、不聚合**任何子技能的记忆。它在"造/改"子技能时，把一套轻量"自我蒸馏能力"焊进该技能；之后子技能**独立蒸馏自己、独立进化**，元技能不再介入。
> 来源：用户原话"让每个技能蒸馏自己"——「老母子」模型的去中心化蒸馏原则。本文件从元技能壳下沉（v3.0.13 瘦身）。

## 为什么去中心化（而非中心化回流）
- **中心化（错误）**：子技能埋反馈钩子 → 元技能聚合蒸馏 → 统一升版。后果：元技能变 25 个技能的记忆垃圾桶，越滚越重，违背"普通技能 E0-E1 轻量"戒律。
- **去中心化（正确）**：每个技能自带蒸馏引擎，自己记、自己萃、自己改。母体只传本事，不替孩子记笔记。

## 赋权三件套（焊入每个子技能，E0 即可承载）
经本元技能造/改的技能，必须自带以下三件套：

**① `evolution-state.json`（技能根目录）** — 子技能自己的进化状态：
```json
{
  "version": "1.0.0",
  "evolution_tier": "E0",
  "usage_count": 0,
  "learned": [],
  "last_distill": null,
  "distill_policy": "self-only"
}
```

**② `usage-log.md`（技能根目录）** — 每次使用的单行记录：
```markdown
# 使用记录
- [YYYY-MM-DD HH:MM] 任务：<一句话> | 反馈：<好/卡点/偏好> | 来源：<会话>
```

**③ SKILL.md 内嵌「自我蒸馏」小节**（复制到子技能 SKILL.md 末尾）：
```markdown
## 自我蒸馏（本技能独立运行，母体不介入）
- **L1 静默记**：每次用完，在 usage-log.md 追加一行（时间戳 + 任务 + 反馈）。
- **L2 周期蒸馏**：累计 ≥3 条同类记录，把规律写进 evolution-state.json 的 learned[]（用户偏好 / 易错点 / 默认参数），usage_count+1。
- **L3 自修补**：规律稳（≥5 条）时，把最优做法固化进本 SKILL.md 对应步骤，并升版本号（E0→E1 等）。
- **铁则**：L1-L2 静默写不需人类确认；L3 改 SKILL.md 必须人类确认（沿用不可变核心第 2 条）。
```

## 元技能的边界（再强调）
- 元技能只负责"造/改时焊入上述三件套"。
- 元技能**永不**读取、调用、汇总、分析子技能的 usage-log.md / evolution-state.json。
- 子技能"越用越好用" = 子技能自己的事，不是元技能的 KPI，也不回流给母体。

## 与 PF 质量门的关系
- 见 PF10（v3.0.2 新增）：造/改技能时核对三件套齐全；auto-test.py（v3.0.7）已纳入 PF10/PF11/PF11f 自动检查。
